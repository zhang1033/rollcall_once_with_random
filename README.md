## 此程序的介绍
>此随机抽取，我会尽量追求小而美，后续可能换成更小的框架。顾名思义，它的主要功能就是随机抽取，除了编辑和检测机制以外，没有任何一个功能与主功能无关。经过十个月的缝缝补补，功能逐渐完善。界面虽然很简陋，但能够让你们易用，就让我很满足了。
    我记得有个转折期就是高倍缩放优化，即高DPI优化。因为在初期版本，窗口是按100%缩放显示的，被系统强行拉大会很糊，强行开高DPI模式会有BUG。于是在11月末进行了高DPI优化，12月初完善。
    我见过其它的点名软件，界面都很糊，强行用高DPI模式界面会错乱。甚至，还有一个软件含有触屏BUG，直接弹窗报错，触屏动不了。我想，或许有人会开发纯win32版本，且有高DPI优化的一类程序，但目前来说，我的会是打开快、界面不糊且不乱的一个。

### 或许有人建议: 
>#### **与其用Python, 不如用electron**

### 这个呢, electron的优点我知道, 就是: 
>1.能够相对自适应界面.
>2.基于Web内核, 会开发网页也会开发APP.
>3.开发成本低.

### ⚠但对于只实现与单一功能有关的功能性程序来说, 或许做不到轻量化. 原因有几些: 
>1.内存占用。我本人看过各electron应用的占用情况，大约300MB起步，对于一个只有单纯的抽取来说，实在太浪费，不如轻量程序省事。
>2.存储占用也有100MB多，包含chromium内核，对于简单功能来说，还是太大了。
>
>3.也是老师最关心的——冷启动速度。electron的天生劣势就是这个，老师等这个很久是会发火的，特别是在课上
